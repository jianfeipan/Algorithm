| 算法             | 类型  | 解决问题   | 适用条件    | 时间复杂度            |
| -------------- | --- | ------ | ------- | ---------------- |
| Dijkstra       | 最短路 | 单源最短路径 | **无负权** | `E log V`        |
| Bellman-Ford   | 最短路 | 单源最短路径 | 可有负权    | `V * E`          |
| Floyd-Warshall | 最短路 | 所有点对   | 小图      | `V³`             |

下面给你一个**一站式总结**：把最常见的图论最短路 / 最小生成树算法（Dijkstra's Algorithm、Prim's Algorithm、Kruskal's Algorithm 等）放在一起对比，讲清楚：

* 解决什么问题
* 什么时候用
* 复杂度
* 核心区别

# 🧠 一、先分清两类问题（最关键）

## 1️⃣ 最短路径（Shortest Path）

👉 从一个点出发，到其他点的**最小代价**

* 代表算法：Dijkstra / Bellman-Ford
* 关注：**路径长度**

---

## 2️⃣ 最小生成树（MST）

👉 用最小代价把所有点**连起来**

* 代表算法：Prim / Kruskal
* 关注：**整体连接成本**

# 🚀 三、逐个讲清楚

---

## 🔵 1. Dijkstra（最常用最短路）

👉 从一个点出发，找最短路径

### 适用

* 边权 ≥ 0
* 单源最短路径

### 复杂度

```id="h62q0g"
O(E log V)
```

### 特点

* 类似 BFS + 优先队列
* 每次选“当前最近的点”

### 典型题

* 网络延迟
* 地图导航
* 最短路径问题

---

## 🔴 2. Bellman-Ford（更通用）

👉 能处理负权

### 适用

* 有负边
* 检测负环

### 复杂度

```id="s0cm9y"
O(V * E)
```

### 特点

* 暴力松弛 V-1 次
* 比 Dijkstra 慢很多

---

## 🟢 3. Prim（MST）

👉 从一个点开始“长树”

### 适用

* 图是连通的
* 想逐步扩展

### 复杂度

* 稀疏图：

```id="2kpq1k"
O(E log V)
```

* 稠密图：

```id="f6g0z2"
O(V²)
```

### 特点

* 很像 Dijkstra（但目标不同）
* 每次选“离当前树最近的点”

---

## 🟡 4. Kruskal（MST）

👉 从全局选最小边

### 适用

* 边少（稀疏图）
* 或边已经排序

### 复杂度

```id="c9p6i1"
O(E log E)
```

### 核心结构

* 并查集（Union-Find）

### 特点

* 全局贪心
* 防止成环

---

# ⚖️ 四、Prim vs Kruskal（重点对比）

| 维度   | Prim         | Kruskal    |
| ---- | ------------ | ---------- |
| 思想   | 从点扩展         | 从边选择       |
| 视角   | 局部           | 全局         |
| 数据结构 | heap / array | union-find |
| 适合   | 稠密图          | 稀疏图        |
| 类似谁  | Dijkstra     | 排序 + 并查集   |

---

# 🔥 五、最容易混淆的点

## ❗ Dijkstra vs Prim

|         | Dijkstra | Prim  |
| ------- | -------- | ----- |
| 目标      | 最短路径     | 最小生成树 |
| dist 含义 | 从起点的距离   | 到树的距离 |
| 更新逻辑    | 路径累加     | 单边最小  |

👉 核心区别一句话：

* Dijkstra：**路径最短**
* Prim：**接入成本最低**

---

# 🎯 六、怎么选算法（面试速判）

## 👉 问题关键词判断

### 出现：

* “最短路径” → Dijkstra
* “所有点连通最小成本” → MST

---

## 👉 MST 再细分

* 点多边密 → Prim（O(n²)版本）
* 边少 → Kruskal

---

## 👉 特殊情况

* 有负权 → Bellman-Ford
* 全点对最短路 → Floyd

---

# 🧠 七、一句话总结（非常重要）

* **Dijkstra**：从一个点出发，找最短路
* **Prim**：从一个点出发，长一棵最便宜的树
* **Kruskal**：从所有边里选最便宜的拼树

---

# 🧩 八、一个直觉类比

想象建通信网络：

* Dijkstra 👉 “从总部到每个城市最便宜路线”
* Prim 👉 “一步步扩展网络，优先接最近的城市”
* Kruskal 👉 “全球选最便宜的电缆，避免形成环”
